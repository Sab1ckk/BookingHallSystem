# Электронная система бронирования конференц-залов

## Описание проекта
Данный проект представляет собой информационную систему для автоматизации процесса бронирования конференц-залов. Система позволяет управлять расписанием мероприятий, отслеживать занятость помещений и предотвращать конфликты при регистрации заявок.

## Логическая структура
База данных проекта включает следующие основные сущности:
* **Rooms** (Залы) — информация о доступных помещениях.
* **Reservations** (Бронирования) — связь между сотрудниками, залами и временем.
* **Employees** (Сотрудники) — данные о персонале компании.
* **Events** (Мероприятия) — детальная информация о проводимых событиях.
* **Users** (Пользователи) — системные аккаунты и права доступа.

## Бизнес-логика и ограничения
В системе реализованы ключевые ограничения для поддержания целостности данных:
- На одну дату и одну комнату может быть назначено не более одного мероприятия.
- Один сотрудник может совершать бронирование только 1 раз в сутки.
- Название мероприятия является обязательным полем.

## Функциональные возможности
### Обработка данных
- **Триггер:** Реализована автоматическая проверка пересечений бронирований.
- **Процедура:** Хранимая процедура для быстрого создания мероприятия.
- **Запросы:** Поиск свободных залов на конкретную дату.

### Пользователи и права доступа
1. **Админ:** Обладает полным доступом к системе.
2. **Организатор:** Имеет право на создание и редактирование событий.
3. **Сотрудники:** Доступны функции просмотра расписания и создания бронирований.
